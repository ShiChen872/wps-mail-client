# WPS Mail 功能差距清单

对照基准：

- **Outlook Web（OWA）**：微软邮箱替换场景下员工日常使用的 Web 能力（非 Outlook 桌面全量）
- **Mailbird**：轻量桌面客户端代表，强项为 **POP3/IMAP/SMTP** 多账号聚合

当前产品定位：WPS 365 公网邮箱专用，**OAuth + OpenAPI**，不走 IMAP（除非 WPS 后续开放且单独做 V2 通道）。

---

## 一、当前已实现（v1.2.0）

| 模块 | 能力 | 备注 |
|------|------|------|
| 登录 | WPS OAuth、token 刷新、主进程保管密钥 | 无需 IMAP 五参数 |
| 邮箱 | 多邮箱切换（主邮箱 + 公共邮箱列表） | `GET /v7/mailboxes` |
| 文件夹 | 系统五夹 + **自定义文件夹树形侧栏** | `GET /v7/mailboxes/{id}/folders`；创建/重命名需 Web |
| 列表 | 分页、**加载更多**、未读数角标 | page_size≤10 |
| 读信 | HTML 正文、**自动/手动已读**、删除、移动 | update/delete/move API + 本地兜底 |
| 写信 | 回复/转发/BCC、**富文本**、**附件上传**、签名(env)、联系人联想 | 附件 API 待 Explorer 复核 |
| 搜索 | 关键字 + **高级搜索**（主题/发件人/正文） | |
| 系统 | 托盘、轮询、**在浏览器中打开 Web 邮箱** | 非推送 |

## 三、对比 Mailbird — 可借鉴 vs 不适用

Mailbird 核心：**任意支持 IMAP/POP3 的邮箱** 聚合到一个桌面壳。与 WPS Mail **路线不同**，但 UX 值得抄。

### 3.1 Mailbird 有、本客户端缺失（体验层可借鉴）

| 功能 | Mailbird | 本客户端 | 建议 |
|------|----------|----------|------|
| 多账号统一收件箱 | 有 | **缺失** | P2：多 WPS 邮箱合并列表 |
| 账户配色/标识 | 有 | **缺失** | P2 |
| 统一收件箱 + 按账号筛选 | 有 | **部分** | 仅下拉切换邮箱 |
| 滑动手势 / 快捷归档 | 有 | **缺失** | P2 |
| Snooze 稍后处理 | 有 | **缺失** | P3，需 API |
| 富文本编辑器 | 有 | **已有** | V1.2 |
| 通讯录整合 | 有 | **缺失** | P1 |
| 第三方集成（WhatsApp 等） | 有 | **不适用** | — |
| 主题/密度定制 | 有 | **缺失** | P2 |

### 3.2 Mailbird 有、本客户端路线不同（不强行对齐）

| 功能 | 说明 |
|------|------|
| POP3 / IMAP / SMTP 手动配置 | Mailbird 核心；WPS 公网邮箱应用 **OAuth**，除非 WPS 公布 IMAP 且企业开启 |
| 代收 Gmail / QQ / 163 | WPS 产品侧「代收第三方」在 **WPS Web 邮箱** 完成，非本客户端 MVP |
| 本地 PST | 无 |
| 每账户独立密码/授权码 | OAuth 统一 WPS 账号 |

### 3.3 若未来要做「Mailbird 式」扩展

| 方案 | 适用场景 |
|------|----------|
| **A. 坚持 WPS OpenAPI** | 微软替换主路径，功能跟 WPS Web 对齐 |
| **B. 增加 IMAP 通道** | 仅当 WPS 365 公网邮箱提供标准 IMAP/SMTP 参数；可做第二账户源 |
| **C. 混合** | 主账户 WPS API，副账户 IMAP（个人 QQ 等）— 复杂度高，V2+ |
