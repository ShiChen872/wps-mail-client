# 试点验收清单（5–10 人）

## 环境准备

- [ ] 开放平台自建应用已开通邮件相关 scope
- [ ] 回调地址 `http://127.0.0.1:38473/callback` 已配置
- [ ] 每位试点用户已分配 WPS 365 邮箱席位
- [ ] 分发 `release/` 安装包或 `npm run dev` 开发包

## 核心路径（对比 Outlook Web / 桌面）

| 场景 | 步骤 | 通过标准 |
|------|------|----------|
| 登录 | 点击「使用 WPS 账号登录」→ 浏览器授权 | 无需 IMAP 参数，30 秒内回到客户端 |
| 冷启动 | 关闭后重新打开 | 3 秒内看到缓存收件箱列表 |
| 收信 | 打开收件箱 → 刷新 | 30 秒内出现最新邮件 |
| 读信 | 点击列表项 | 正文与附件名正确显示 |
| 附件 | 点击附件 → 另存为 | 文件可打开 |
| 发信 | 写邮件 → 发送 | 对方 Web 邮箱可收到 |
| 搜索 | 输入关键字回车 | 返回相关邮件 |
| 通知 | 等待约 2 分钟或收到新邮件 | 托盘/tooltip 未读数更新（可选系统通知） |

## 主观评价

- [ ] 受访者认为可替代 **≥80%** 日常 Outlook 收发（不含日历/会议）
- [ ] 记录主要缺口（如：标记已读、删除、规则）供 V1.1 排期

## 打包命令

```bash
set ELECTRON_MIRROR=https://cdn.npmmirror.com/binaries/electron/
npm run dist
```

产物：`release/WPS Mail Setup *.exe` 与 portable 版本。
